---
title: 'Update Activity'
api: 'PATCH /sequences/:sequenceId/activities/:activityId'
description: 'Update activity content, scheduled time, or approval status'
---

## Path Parameters

<ParamField path="sequenceId" type="integer" required>
  Sequence ID
</ParamField>

<ParamField path="activityId" type="integer" required>
  Activity ID
</ParamField>

## Body Parameters

<ParamField body="subject" type="string">
  Updated email subject (for outgoing_email activities)
</ParamField>

<ParamField body="body" type="string">
  Updated email body HTML (for outgoing_email activities)
</ParamField>

<ParamField body="due_at" type="string">
  Updated scheduled time (ISO 8601)
</ParamField>

<ParamField body="status" type="string">
  Update status: `pending`, `approved`, `rejected`
</ParamField>

<ParamField body="rejection_reason" type="string">
  Rejection reason (required when status is `rejected`)
</ParamField>

<RequestExample>
```json Update Content

  "body": "<p>Updated content...</p>",
  "due_at": "2025-11-11T10:00:00Z"

```

```json Approve Activity
}
```

```json Reject Activity
{
  "status": "rejected",
  "rejection_reason": "Email tone is too formal, needs to be more casual and friendly"
}
```
</RequestExample>

## Behavior Notes

### Content Updates
- Users can iterate on drafts multiple times before approving
- Each update modifies the associated `email_outgoing` record
- Activity remains in `pending` status

### Approval
- Setting `status="approved"` schedules the email for sending
- Email status changes to `scheduled`
- Triggers Workflow #6 (Email Sending) with rate limiting

### Rejection
- Setting `status="rejected"` cancels the current draft
- Automatically triggers Workflow #4 (AI Next Action) to generate new alternative
- Rejection reason is included in AI context for better results

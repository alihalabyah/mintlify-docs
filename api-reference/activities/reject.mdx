---
title: 'Reject Draft Email'
api: 'POST /activities/:id/reject'
description: 'Reject a draft email and trigger AI to generate a new alternative'
---

## Path Parameters

<ParamField path="id" type="integer" required>
  Activity ID
</ParamField>

## Body Parameters

<ParamField body="reason" type="string">
  Optional rejection reason for tracking and AI learning
</ParamField>

<RequestExample>
```json
{
  "reason": "Email tone is too formal, needs to be more casual and friendly"
}
```
</RequestExample>

## Response

<ResponseField name="data" type="object">
  Rejected activity object
  
  <Expandable title="Rejected Activity Object">
    <ResponseField name="id" type="integer" required>
      Activity ID
    </ResponseField>
    
    <ResponseField name="sequence_lead_id" type="integer" required>
      Sequence lead ID
    </ResponseField>
    
    <ResponseField name="activity_type" type="string" required>
      Activity type
    </ResponseField>
    
    <ResponseField name="status" type="string" required>
      Status (will be `rejected`)
    </ResponseField>
    
    <ResponseField name="is_approved" type="boolean">
      Approval flag (will be false)
    </ResponseField>
    
    <ResponseField name="rejection_reason" type="string">
      User-provided rejection reason
    </ResponseField>
    
    <ResponseField name="created_at" type="string" required>
      Creation timestamp
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseExample>
```json
{
  "data": {
    "id": 321,
    "sequence_lead_id": 1001,
    "activity_type": "outgoing_email",
    "status": "rejected",
    "is_approved": false,
    "outgoing_email_id": 9003,
    "rejection_reason": "Email tone is too formal, needs to be more casual and friendly",
    "created_at": "2025-11-10T15:00:00Z"
  },
  "metadata": {},
  "pagination": null
}
```
</ResponseExample>

<Note>
  After rejection, Workflow #4 (AI Next Action) is automatically triggered to generate a new draft email. The rejection reason is included in the AI context to help generate a better alternative.
</Note>
